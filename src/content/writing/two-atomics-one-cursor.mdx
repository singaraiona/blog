A design rewrite that took a month and ships today as 0.2.5. The commit messages from this last week tell the story more honestly than I could:

```
two-atomics schema
two-atomics cursor schema
spin on head overtake
do not allow head overtake tail
fixed flags logic in a head
```

The "two atomics" name is for the cursor at each end of the channel. A cursor is now exactly:

```rust
struct Cursor<T> {
    index: AtomicUsize,
    block: AtomicPtr<Block<T>>,
}
```

`index` carries the sequence position with one flag bit reserved at the top. `block` points at the current `Block<T>` of slots. The channel has two of these, padded onto separate cache lines:

```rust
struct Channel<T> {
    tail: CachePadded<Cursor<T>>,
    head: CachePadded<Cursor<T>>,
    receivers: AtomicUsize,
    senders: AtomicUsize,
}
```

A block is fixed-size:

```rust
const ROUND: usize = 64;
const BLOCK_SIZE: usize = ROUND - 1;

struct Block<T> {
    next: AtomicPtr<Block<T>>,
    slots: [Slot<T>; BLOCK_SIZE],
}
```

`head & BLOCK_SIZE` gives the slot inside the current block. `head / ROUND` gives the block-sequence number. The producer writes a slot, advances `tail.index`. The consumer reads a slot, advances `head.index`. When either steps past `BLOCK_SIZE`, it follows `block.next` (or allocates a new block, on the producer side, and links it).

The flag bit lives at `1 << 63`. It serves two purposes, depending on which cursor reads it:

```rust
const CLOSED_FLAG:  usize = 1 << 63;
const CROSSED_FLAG: usize = CLOSED_FLAG;
```

On `tail.index`, the bit is `CLOSED_FLAG`: producers see it and stop. On `head.index`, the same bit acts as `CROSSED_FLAG`: it tells the consumer it has stepped onto a block the producer no longer points at. The reused bit is the difference between "spin once and recover" and "lock to be safe".

The rule that kept me sane during this rewrite was: "do not allow head overtake tail". It is one line in the spec, two pages of edge cases in the implementation, and exactly one assertion in the tests. Every time I broke it, the tests caught it within a minute.

0.2.5 is on crates.io tonight. I am not going to push more on this crate for a while. It does what it set out to do, and the API has been stable across three minor versions. There is a moment in any library's life where the right thing is to leave it alone. I think this one is there.
